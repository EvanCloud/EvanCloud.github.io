jenkins添加远程主机

Jenkins 添加远程的SSH主机，并在远程的主机上面执行命令或者脚本。这个时候就需要为远程的主机进行注册,配置步骤如下：

1. 先在 Jenkins 中添加 [ 凭据 ]

​ 进入 Jenkins Web 界面后点击 [ 凭据 ] --> [ 添加凭据 ]，根据提示填写凭据的 名称 和 私钥（也可使用密码登录的方式） 。私钥可以是自己生成的，也可使用 Jenkins 自动生成的，Jenkins 生成的需要到以下目录获取 :

注意：如果是使用 Key 的这种方式需要将公钥提前放入远程主机的对应目录下：

2.配置远程主机信息

​ 进入 [ 系统管理 ] ---> [ 系统设置 ] ，找到 [ SSH remote hosts ] 模块，依次填入 :

  • Hostname -- 目标主机地址
  • Port -- SSH 开放的端口，默认是22
  • Credentials -- 选择对应的凭据

关于SCP 配置后不能使用的问题

​ 需要将目标主机的信息写入到 Jenkins 的【 /var/lib/jenkins/.ssh/known_hosts 】 文件中，不然第一次Jenkisn 无法成功SCP文件到目标主机，因为在第一次的 SCP 过程中有一个需要输入 “YES” 确认反馈。现在Jenkins 主机上ssh 登录一下目标主机，然后退出后将 [ ~/.ssh/authorized_keys ] 的内容写入到Jenkins的【 /var/lib/jenkins/.ssh/known_hosts 】 内即可。

Grafana展示CLoudWatch监控项

一、添加数据来源

​ 在garfana 的Configuration 中选择Data sources 并点击 "Add data source" 来添加数据来源，注意再garfana的现版本中已经内建支持 AWS CloudWatch 为数据来源，我们只需要在 Type 中选择 CloudWatch。

​ 关于认证身份有三种方法，分别为：

  • ARN
  • Credentials file
  • Access & secret key

注意：使用文件认证和使用access &secret key 本质上都是使用 Access & secret key来作为认证条件，只不过一种是在grafana的Web 界面上直接填写，一种是在garfan的部署服务器的指定位置写入Access & secret key 信息

示例：

[参考文档](<a href="超链接地址" target="_blank">超链接名</a>)

二、关于AWS上的权限设置

​ 我们配置了Access & secret key后连接测试显示正确，但是在控制面板却拉取不到数据，这往往是我们的用户策略上设置的不正确。

不论是我们用来拉取CloudWatch的用户，还是赋予EC2实例上的角色必须拥有以下的策略：

  • cloudwatch
    • List: ListMetrics
    • Read: GetMetricData 、GetMetricStatistics
